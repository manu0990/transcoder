# transcoder
This repo contains services that helps to transcode files in various formats

## Swagger API docs

Swagger UI is exposed at `http://localhost:3000/api-docs` when the server is running. The OpenAPI spec is generated by `swagger-jsdoc` from route source files and then served with `swagger-ui-express`.

Current setup:

- Swagger config lives in [src/swagger.ts](/d:/codes/Transcoder/server/src/swagger.ts).
- The docs UI is mounted in [src/server.ts](/d:/codes/Transcoder/server/src/server.ts).
- Route-level OpenAPI comments are read from files in `src/routes/*.ts`.

How feature developers should use it:

1. Add the Express handler in `src/routes/`.
2. Add an `@openapi` JSDoc block above the handler export.
3. Describe the route path, method, summary, request body, params, and responses.
4. Register the handler in [src/index.ts](/d:/codes/Transcoder/server/src/index.ts) or the relevant router.
5. Start the server and verify the endpoint appears in `/api-docs`.

Example pattern:

```ts
/**
 * @openapi
 * /videos/{id}:
 *   get:
 *     summary: Fetch a video by id
 *     tags:
 *       - Videos
 *     parameters:
 *       - in: path
 *         name: id
 *         required: true
 *         schema:
 *           type: string
 *     responses:
 *       200:
 *         description: Video details
 */
export const getVideo = (req: Request, res: Response) => {
  res.status(200).json({ id: req.params.id });
};
```

Notes:

- Use the exact route path in the OpenAPI block, not the file name.
- Keep docs close to the handler so API behavior and documentation change together.
- If you build the project, Swagger can also read generated files from `dist/routes/*.js`.
